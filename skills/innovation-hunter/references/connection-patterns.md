# Connection Patterns

## 四大连接策略

### 1. 移植 (Transplant)
把X领域的思想应用到Y领域。

| 从 | 到 | 潜力 |
|----|----|------|
| Agent记忆机制 | 卫星网络临时存储 | 高 |
| 多模态对比学习 | 北斗+视觉定位融合 | 高 |
| RLHF对齐 | 通信协议自适应调优 | 中 |
| CoT推理链 | 卫星故障诊断推理 | 中 |

### 2. 组合 (Combine)
两个成熟技术叠加产生新产品。

| A | B | 潜力 |
|---|---|------|
| VLM | 卫星图像 | 遥感自动解读 |
| Agent框架 | 通信协议 | 自适应网络优化 |
| 多模态Agent | 北斗定位 | 视觉增强定位 |
| 多Agent协作 | LEO星座 | 星间协同组网 |

### 3. 类比 (Analogy)
X的什么问题像Y？

- Agent规划 ≈ 卫星路由规划
- 多模态对齐 ≈ 多源定位融合
- Agent记忆 ≈ 卫星过顶窗口调度
- Tool-use Agent ≈ 卫星载荷调用

### 4. 反转 (Reverse)
反过来想。

- 不是"AI控制卫星" → 而是"卫星为AI提供独特数据"
- 不是"用Agent解析通信" → 而是"Agent重构通信协议设计"
- 不是"多模态理解卫星图" → 而是"卫星视角增强多模态推理"

## 假阳性过滤表

| 想法 | 判定 | 原因 |
|------|------|------|
| 用Agent做卫星数据解析 | ❌ 假阳性 | 只是方法迁移，无真实挑战 |
| 多模态+北斗=地图叠加图像 | ❌ 假阳性 | 显而易见，无创新壁垒 |
| VLM辅助遥感图像理解 | ✅ 真连接 | 有技术挑战（云遮挡、分辨率） |
| 多Agent模拟卫星组网 | ✅ 真连接 | 新颖且有实际价值 |
| Agent优化北斗信号切换 | ✅ 真连接 | 真实痛点，算法有价值 |
